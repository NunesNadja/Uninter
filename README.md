# Portfolio Pessoal - Nadja Kelly de Oliveira

Projeto desenvolvido para a disciplina de Fundamentos da Programacao Web, utilizando apenas:
- HTML5
- CSS3 (sem frameworks)
- JavaScript puro (sem frameworks)

## Visao geral
Este portfolio apresenta informacoes profissionais e academicas em 4 paginas interligadas por menu fixo:
1. Sobre mim
2. Formacao
3. Portfolio
4. Contato

## Estrutura de pastas e arquivos
- `index.html`: pagina inicial (Sobre mim).
- `pages/education.html`: pagina Formacao.
- `pages/portfolio.html`: pagina Portfolio/Experiencias.
- `pages/contact.html`: pagina Contato com formulario validado em JavaScript.
- `css/base.css`: variaveis globais, reset e tipografia.
- `css/layout.css`: estrutura geral da pagina (header, main, footer).
- `css/components.css`: menu, cards, hero, botoes, foto e formulario.
- `css/animations.css`: animacoes de entrada e destaque.
- `css/responsive.css`: regras responsivas para mobile.
- `js/script.js`: interacoes de menu, tema, validacao e simulacao de envio.
- `assets/image.png`: foto de perfil.

## Funcionalidades implementadas
- Menu visivel em todas as paginas.
- Tema claro personalizado (paleta rosa) e tema escuro.
- Persistencia de tema com `localStorage`.
- Menu responsivo (botao hamburguer no mobile).
- Formulario de contato com validacao:
  - Nome obrigatorio
  - E-mail obrigatorio com formato valido
  - Mensagem obrigatoria
- Simulacao de envio com limpeza dos campos e mensagem de sucesso.

## Como executar localmente
1. Abra a pasta `Uninter` no VS Code.
2. Abra o arquivo `index.html` no navegador (ou use a extensao Live Server).

## Publicacao no GitHub Pages
1. Crie um repositorio publico no GitHub.
2. Envie os arquivos do projeto para a branch `main`.
3. No GitHub, acesse `Settings` > `Pages`.
4. Em `Build and deployment`, selecione:
   - `Source`: `Deploy from a branch`
   - `Branch`: `main` e pasta `/root`
5. Aguarde a publicacao e copie o link final.

Exemplo de URL:
- `https://seu-usuario.github.io/nome-do-repositorio/`

## Checklist para entrega (PDF)
- Link do site publicado.
- Link do repositorio publico no GitHub.
- Prints das paginas com a URL visivel no navegador.
- Codigo-fonte no PDF (apenas se exigido pela plataforma).
